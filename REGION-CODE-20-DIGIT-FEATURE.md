# 地区编码20位数字功能说明

## 功能概述

地区管理模块现已支持**1-20位数字**的地区编码，相比之前的6位数字限制，提供了更大的灵活性和扩展性。

## 主要变更

### 1. 前端验证规则更新

#### 表单验证
```javascript
// 修改前：只允许6位数字
{ pattern: /^[0-9]{6}$/, message: '地区编码必须为6位数字', trigger: 'blur' }

// 修改后：允许1-20位数字
{ pattern: /^[0-9]{1,20}$/, message: '地区编码必须为1-20位数字', trigger: 'blur' }
```

#### 用户提示更新
```vue
<!-- 修改前 -->
<div class="form-tip">地区编码为6位数字，如：110000</div>

<!-- 修改后 -->
<div class="form-tip">地区编码为1-20位数字，如：110000、12345678901234567890</div>
```

### 2. 导入功能说明更新

#### 导入要求
- **地区编码**：1-20位数字，唯一标识
- **示例格式**：
  - 传统6位：110000（北京市）
  - 扩展8位：11000001（北京市朝阳区）
  - 扩展12位：110000010001（北京市朝阳区某街道）
  - 扩展20位：12345678901234567890（测试编码）

### 3. 数据库支持

#### 字段长度
```sql
-- 地区编码字段
region_code VARCHAR(20) NOT NULL COMMENT '地区编码'

-- 父级编码字段
parent_code VARCHAR(20) DEFAULT '' COMMENT '父级编码'
```

#### 索引支持
- `region_code` 字段已建立唯一索引
- `parent_code` 字段已建立普通索引
- 支持20位数字的快速查询

## 故障排除

### 1. 常见问题

#### Q: 为什么无法输入超过6位的编码？
A: 检查前端验证规则是否已更新，确保正则表达式为 `/^[0-9]{1,20}$/`

#### Q: 导入20位编码失败怎么办？
A: 检查数据库字段长度是否为VARCHAR(20)，确保后端支持

#### Q: 现有6位编码是否受影响？
A: 完全不受影响，向后兼容
