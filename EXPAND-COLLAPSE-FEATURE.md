# 地区管理展开收起功能说明

## 功能概述

地区管理模块现在支持每个父级地区独立展开收起的功能，用户可以单独控制每个省级、市级地区的展开状态，实现更灵活的数据浏览体验。

## 功能特性

### 1. 独立展开控制
- **省级地区**：可以展开查看下属市级地区
- **市级地区**：可以展开查看下属区级地区
- **区级地区**：没有子级，不显示展开按钮
- **独立操作**：每个父级地区的展开状态互不影响

### 2. 智能按钮显示
- **展开状态**：▶ 箭头向右，表示可以展开
- **收起状态**：▼ 箭头向下，表示已经展开
- **占位符**：区级地区前面显示占位符，保持对齐

### 3. 状态保持
- **操作保持**：新增、编辑、删除操作后保持展开状态
- **刷新保持**：刷新数据后恢复之前的展开状态
- **查询保持**：查询操作后保持展开状态

- **未展开**：`<ArrowRight />` 箭头向右
- **已展开**：`<ArrowDown />` 箭头向下

## 技术实现

### 1. 数据结构
```javascript
// 展开状态记录
const expandedKeys = ref(new Set())

// 检查节点是否已展开
const isExpanded = (id) => {
  return expandedKeys.value.has(id)
}
```

### 2. 展开收起逻辑
```javascript
// 处理展开/收起切换
const handleToggleExpand = async (row) => {
  if (isExpanded(row.id)) {
    // 收起节点
    expandedKeys.value.delete(row.id)
    row.children = []
  } else {
    // 展开节点
    const response = await request.get(`/api/region/children/${row.id}`)
    const children = response.data.data || []
    
    // 为子节点添加 hasChildren 属性
    children.forEach(child => {
      child.hasChildren = child.level < 3
    })
    
    row.children = children
    expandedKeys.value.add(row.id)
  }
}
```

### 3. 状态保持机制
```javascript
// 在数据操作后保持展开状态
const currentExpandedKeys = new Set(expandedKeys.value)
fetchRegionList()
expandedKeys.value = currentExpandedKeys
```

## 使用流程

### 1. 展开省级地区
1. 在省级地区前面找到展开按钮（▶）
2. 点击展开按钮
3. 按钮变为收起状态（▼）
4. 显示下属市级地区列表

### 2. 展开市级地区
1. 在市级地区前面找到展开按钮（▶）
2. 点击展开按钮
3. 按钮变为收起状态（▼）
4. 显示下属区级地区列表

### 3. 收起地区
1. 点击已展开地区的收起按钮（▼）
2. 按钮变为展开状态（▶）
3. 隐藏下属地区列表

## 功能优势

### 1. 用户体验
- **直观操作**：每个父级都有独立的展开按钮
- **状态清晰**：箭头方向明确表示展开状态
- **操作便捷**：点击按钮即可展开收起

### 2. 性能优化
- **按需加载**：只在展开时加载子级数据
- **状态保持**：操作后保持展开状态，避免重复操作
- **内存友好**：收起时清空子级数据

### 3. 数据一致性
- **实时更新**：展开时获取最新数据
- **状态同步**：前端状态与后端数据保持一致
- **错误处理**：加载失败时显示友好提示

## 注意事项

### 1. 数据加载
- 展开操作会发起网络请求获取子级数据
- 如果网络较慢，请耐心等待
- 加载失败会显示错误提示

### 2. 状态管理
- 展开状态只在当前页面有效
- 刷新页面后会重置展开状态
- 建议在操作完成后手动收起不需要的展开项

### 3. 性能考虑
- 避免同时展开大量地区
- 收起不需要的地区以释放内存
- 定期刷新数据保持信息最新

## 扩展功能

### 1. 批量操作

新的展开收起功能为地区管理提供了更好的用户体验，每个父级地区都可以独立控制展开状态，操作更加直观和便捷。通过智能的状态保持机制，用户在操作数据后无需重新展开，大大提升了工作效率。该功能既满足了数据浏览的需求，又优化了系统性能，是地区管理系统的重要改进。
